5。通知

通知理解：用到Notification类，而改类继续使用构造器来构造该类， 图标必须设置，如果要自定义通知需要用到RemoteViews远程视图类，如果需要设置单机以后的意图PendingIntent的远程意图，在构建到构建器当中

    public void notifi(View v)
    {
        Notification.Builder builder = new Notification.Builder(context);
        builder.setAutoCancel(true);// 自动取消
        builder.setContentTitle("内容标题");
        builder.setContentText("内容格式");
        builder.setTicker("班级重要通知");
        builder.setSmallIcon(R.drawable.emoji_080); // 必须设置，不然会谈不出来
        builder.setDefaults(Notification.DEFAULT_SOUND);
        builder.setWhen(System.currentTimeMillis());// 设置通知时间

        /*
         * 自己定义一个通知界面 1.定义一个远程视图,不是在该应用中展示，而是在远程的应用中展示
         * 值要定义了视图。Remoteviews类可以方便的设置视图里面的参数
         */
        RemoteViews remoteViews = new RemoteViews(getPackageName(), R.layout.notifi_layout);
        remoteViews.setTextViewText(R.id.notifitext_tv, "你有一条QQ消息");
        builder.setContent(remoteViews);

        /*
         * 设置单击通知以后的意图。 这里模拟的是打电话
         */
        Intent intent = new Intent();
        intent.setAction(Intent.ACTION_CALL);
        intent.setData(Uri.parse("tel://10086"));
        /**
         * 通知发送延迟意图
         */
        PendingIntent pendingIntent = PendingIntent.getActivity(context, 0, intent, PendingIntent.FLAG_CANCEL_CURRENT);
        builder.setContentIntent(pendingIntent);

        Notification notification = builder.build();
        // 通知管理器发布 ... 获取到通知服务
        NotificationManager notificationManager = (NotificationManager) getSystemService(Context.NOTIFICATION_SERVICE);
        // 发送通知
        notificationManager.notify(1, notification);

    }

  • LayoutInflater类

    包 package android.view

    一个activity在使用onCreate()方法调用setContentView()方法加载布局文件的时候，就会调用LayoutInflater类的实列化定义在布局对象中的每一个view对象。实际如View layoutView = LayoutInflater.from(context).inflate(R.layout.toast_layout, null);

  • Context类
    1、它描述的是一个应用程序环境的信息，即上下文。
    2、该类是一个抽象(abstract class)类，Android提供了该抽象类的具体实现类(后面我们会讲到是ContextIml类)。
    3、通过它我们可以获取应用程序的资源和类，也包括一些应用级别操作，例如：启动一个Activity，发送广播，接受Intent
    信息 等。。

里面封装了许多抽象类，如获取服务管理器等等。
